## [2026-01-29] Task 2: 변신 장면 생성 예상 이슈

### 기술적 제약사항

#### 1. AI 비디오 생성 도구의 변신 한계
**이슈**: 현재 AI 비디오 도구들은 "변신"을 주 기능으로 설계되지 않음
- Pika Labs: 변신 효과 있으나 단순화 경향
- Runway Gen-4: 모션 생성 중심, 복잡한 형태 변환 어려움
- Kling AI: 문서 부족으로 품질 예측 불가

**영향**: 트랜스포머 스타일 완벽 재현은 어려울 수 있음

**완화 전략**:
- Pika Pikaffects로 추상적 변신 (파티클 효과)
- Plan B 멀티클립 조합으로 중간 과정 감추기
- 기대치 조정 (영화 수준 아닌 "인식 가능한 변신")

#### 2. 클립 길이 제약
**이슈**: 대부분 도구가 5-10초 짧은 클립 생성
- 변신 장면 2초 이상 필요
- 짧은 클립에서 복잡한 변신 표현 어려움

**완화 전략**:
- Pika Pikaframes 활용 (5-25초 시퀀스)
- 프롬프트에 "slow transformation" 명시
- 멀티클립 조합으로 총 길이 확보

#### 3. 중간 프레임 품질 문제
**이슈**: 변신 중간 과정에서 기괴한 형태 생성 가능
- 코뿔소도 트럭도 아닌 애매한 형태
- 해부학적/물리적으로 불가능한 형태

**완화 전략**:
- 파티클/플래시 효과로 중간 과정 감추기
- 빠른 전환으로 중간 프레임 노출 최소화
- Pika 2.5 개선된 물리 엔진 활용

### 프롬프트 엔지니어링 난이도

#### 4. 프롬프트 품질 의존성
**이슈**: 결과물이 프롬프트 작성 능력에 크게 의존
- 미묘한 표현 차이로 결과 크게 달라짐
- 최적 프롬프트 찾기까지 많은 시행착오 필요

**완화 전략**:
- 10+ 프롬프트 변형 사전 준비
- Iteration 예산 확보 (30-40회 생성)
- 커뮤니티 예제 참고

#### 5. 도구별 프롬프트 문법 차이
**이슈**: Pika, Runway, Kling 각각 다른 프롬프트 스타일 선호
- 한 도구에서 성공한 프롬프트가 다른 도구에서 실패 가능

**완화 전략**:
- 도구별 공식 가이드 숙지
- 각 도구에 맞춘 프롬프트 변형 준비

### 비용 및 시간 리스크

#### 6. 무료 크레딧 소진 가능성
**이슈**: 테스트 + iteration으로 무료 크레딧 빠르게 소진
- 예상 생성 횟수: 30-40회
- 각 도구 무료 티어 제한 불명확

**완화 전략**:
- 우선순위 도구부터 테스트 (Pika 우선)
- 유료 전환 예산 확보 ($12-24/월)
- 효율적 iteration (명확한 품질 기준)

#### 7. 학습 곡선
**이슈**: 사용자가 AI 비디오 도구 초보
- 각 도구 인터페이스 학습 필요
- 프롬프트 작성 기술 습득 필요
- CapCut 편집 기술 필요 (Plan B 시)

**완화 전략**:
- 상세한 실행 계획 제공
- 단계별 체크리스트
- 공식 튜토리얼 링크 제공

### 품질 관련 리스크

#### 8. 기대치 불일치
**이슈**: 사용자가 영화 수준 품질 기대 가능
- 현재 AI 도구는 트랜스포머 영화 수준 불가
- "변신"의 정의가 모호할 수 있음

**완화 전략**:
- 명확한 품질 기준 정의 (3단계 레벨)
- 현실적 기대치 설정
- Plan B 시나리오 사전 합의

#### 9. 캐릭터 일관성 유지 어려움
**이슈**: 변신 전후 캐릭터 특징 소실 가능
- 코뿔소 뿔, 얼굴 특징 사라짐
- 트럭 디자인이 무쏘와 다를 수 있음

**완화 전략**:
- 프롬프트에 연결 요소 명시
- 색상 일관성 유지
- 참조 이미지 활용 (가능한 도구)

### 해결되지 않은 질문

#### 10. 실제 생성 품질 불확실성
**이슈**: 계정 생성 전까지 실제 품질 확인 불가
- 이론적 전략이 실제로 작동할지 미지수
- 도구별 실제 성능 차이 불명확

**대응 계획**:
- Phase 1 테스트로 빠른 검증
- 결과 기반 전략 수정 준비
- 여러 도구 병행 테스트

#### 11. 한국어 지원 불확실성
**이슈**: 일부 도구의 한국어 프롬프트 지원 불명확
- 영어 프롬프트 필요 가능성
- 번역 과정에서 의미 손실 가능

**대응 계획**:
- 영어 프롬프트 우선 사용
- 필요 시 한국어 테스트
- 프롬프트 번역 품질 검증

### 모니터링 필요 사항
- 각 도구의 실제 무료 크레딧 한도
- 생성 품질의 일관성
- 프롬프트 성공률
- 사용자 학습 속도
- Plan B 전환 시점 판단
